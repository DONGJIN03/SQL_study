## TOP-N쿼리 vs ORDER BY + LIMIT

#### 1. 개념 요약
- TOP-N 쿼리 
  특정 기준으로 정렬된 결과에 상위 N개 행을 가져옴
  → 윈도우 함수 기반 TOP-N 쿼리는 그룹별 상위 N개를 뽑을때 자주 사용

- ORDER BY + LIMIT
  전체 결과를 정렬한 뒤, 상위 N개만 잘라서 반환  
  → 단순히 상위 N개를 가져오므로 그룹별 상위 결과는 얻지 못함

#### 2. 언제 사용하는가 (판단 기준)
- 전체 TOP-N 필요 → TOP-N쿼리, ORDER BY + LIMIT 둘다 사용 가능
- 그룹별 TOP-N 필요 → TOP-N 쿼리

#### 3. 헷갈리기 쉬운 비교 포인트
- ORDER BY + LIMIT: 전체 결과에서 상위 N개 뽑을때
- TOP-N 쿼리(윈도우 함수 포함): 그룹별/조건별 상위 N개
